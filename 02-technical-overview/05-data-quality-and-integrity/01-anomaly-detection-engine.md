# Anomaly Detection Engine

We are running continuous tests to filter out cheater

## User Validation

*   Users must have two-factor authentication (2FA) enabled.
*   Users must log in with a valid email address from a known domain.
*   Users' IP addresses or phone numbers should not overlap with other accounts.

## Measurement Integrity

*   Locations should not be repeated during the measurement period.
*   dB(A) values should vary throughout the measurement period.
*   Measurements should not be taken in prohibited locations, such as bodies of water.

## GPS Spoofing Prevention

*   User speeds should not exceed 12 km/h (7.46 m/h).
*   The distance between recorded locations should be consistent with the calculated speed.
*   Locations should not form a straight line, which could indicate artificial generation.

## Timing Consistency

*   The timing between each location update should be consistent with the expected periodic reports from the sensor.

## Hexagon Variations

*   Noise levels should be consistent with previously measured levels in the same hexagons. Large deviations may indicate a problem.

## Preventive Measures

*   Additional actions may be taken if Silencio detects user behaviors inconsistent with typical human behavior.

## Community Flagging System

*   Silencio empowers users to anonymously flag suspicious activity, adding an extra layer of security.

In addition, Silencio employs undisclosed detection measures, including AI-driven tools, to identify potential anomalies. These safeguards—together with a strong community and active moderators—help maintain a healthy environment where genuine users can fully enjoy Silencio.
